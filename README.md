# 作品アクセス解析ダッシュボード

Web小説の日次アクセス数（PV・スキ・ブックマークなど）をGoogleスプレッドシートに記録し、ブラウザで可視化する **単一HTMLファイル** のダッシュボードです。

- 📊 主要指標カード（前日増減つき／一部は「1話あたり平均」も表示）
- 📈 推移グラフ（PV・スキ・エンゲージメント・質の指標）＋表示期間の切り替え
- 🗂 スプレッドシートの **タブ＝作品** として複数作品をセレクタで切り替え
- 🔌 **APIキー不要**・**インストール不要**。`index.html` をダブルクリックで動きます
- 🔒 データはあなたのブラウザとGoogleの間で完結（サーバーに送信しません）

---

## クイックスタート

### 1. テンプレートを用意する
同梱の [`template.xlsx`](template.xlsx) を **Googleスプレッドシートにアップロード** して開きます
（Googleドライブ →「新規」→ ファイルのアップロード → 開いたら「Googleスプレッドシートとして保存」）。
数式（PV合計・各種割合・前日増減など）は入力済みです。

- 1つのタブ（シート）＝1作品です。作品を増やすときは **タブを右クリック →「複製」** して使います。
- 毎日、**入力列**（下表の「入力」）だけを1行追記していけばOK。数式列は自動計算されます。

### 2. スプレッドシートを共有可能にする ⚠ 重要
右上 **「共有」→「一般的なアクセス」→「リンクを知っている全員」→「閲覧者」** に設定します。
（これをしないとダッシュボードからデータを読み込めません）

### 3. ダッシュボードを開く
[`index.html`](index.html) をダブルクリックで開きます（またはGitHub Pages等のURLで開く）。

### 4. スプレッドシートを登録する
画面上部 **「⚙ データソース設定」** を開き、

1. スプレッドシートのURLを貼って **「URLからID更新」**
2. 各作品タブをクリックするとURL末尾が `#gid=1234567` に変わります。その数字を作品ごとに改行またはカンマ区切りで入力し **「タブ一覧を更新」**

以降は **「🔄 最新を取得」** で最新データを読み込みます。設定はブラウザ（localStorage）に保存されるので次回以降は自動で読み込まれます。

> まず動きだけ試したい場合は、手順2以降をとばして、同梱の [`sample-data.csv`](sample-data.csv) を「⚙ データソース設定」内のCSV欄にドラッグ＆ドロップしてください。

## 入力の手間を減らす（任意）

毎日の入力を手作業でやる代わりに、筆者は次のようにしています（あくまで一例です）。

「Obsidian Web Clipper」というChrome用の拡張機能で「作品編集」ページをクリップし（「作品の反響」の表示を「年」にしてからクリップ）、そのmdファイルをGemini（3.5 Flash）でMarkdownの表形式に変換して貼り付けています。
プロンプトは以下のとおり。複数ファイルをまとめて変換できます。

```
添付のファイルを分析して以下のデータをMarkdownの表形式で出力して。
「話」や「文字」などの単位テキストは除外し数字のみにするが、「%」は残す。
データ取得日 | データ取得時間 | 作品タイトル | 作品トップPV | エピソードPV | スキ | ブックマーク | コメント | 推薦レビュー | 読了率 | 公開話数 | 公開文字数
```

## スプレッドシートの様式（列の定義）

**列の見出し（ヘッダ名）で対応付けます**。列の順番が違っても、見出しさえ一致していれば読み込めます。逆に、見出し名が1文字でも違うとその列は空欄扱いになります。

| 列見出し | 種別 | 内容 / 数式（2行目の例。3行目以降は行番号をずらす） |
|---|---|---|
| データ取得日 | 入力 | 例: `6月15日(月)`（記録した日） |
| データ取得時間 | 入力 | 例: `6:05`（任意） |
| 作品タイトル | 入力 | 作品名。**空欄の行は読み飛ばされます** |
| 作品トップPV | 入力 | 作品トップページのPV |
| エピソードPV | 入力 | 各話の合計PV |
| スキ | 入力 | スキ（いいね）数 |
| ブックマーク | 入力 | ブックマーク数 |
| コメント | 入力 | コメント数 |
| 推薦レビュー | 入力 | 推薦レビュー数 |
| 読了率 | 入力 | 例: `86%` |
| 公開話数 | 入力 | 公開済みの話数 |
| 公開文字数 | 入力 | 公開済みの合計文字数 |
| PV合計 | 数式 | `=D2+E2`（作品トップPV＋エピソードPV） |
| 作品トップPVの割合 | 数式 | `=D2/M2` （%表示） |
| スキ/エピソードPV | 数式 | `=F2/E2` （%表示） |
| エピソードPV/話 | 数式 | `=E2/K2` |
| スキ/話 | 数式 | `=F2/K2` |
| 平均文字数 | 数式 | `=L2/K2` |
| エピソードPV前日増減 | 数式 | `=E3-E2`（3行目以降。**最初のデータ行は空欄**） |
| スキ前日増減 | 数式 | `=F3-F2`（3行目以降。**最初のデータ行は空欄**） |

> 列の対応（A=データ取得日, B=データ取得時間, C=作品タイトル, D=作品トップPV, E=エピソードPV, F=スキ, G=ブックマーク, H=コメント, I=推薦レビュー, J=読了率, K=公開話数, L=公開文字数, M=PV合計 …）。`template.xlsx` にはこれらの数式がすでに入っています。

数値のカンマ区切り（`1,234`）や末尾の `%` は自動で除いて解釈します。

## オフラインで試す / バックアップ運用

ネットに繋がなくても、CSVファイルをドラッグ＆ドロップすれば1作品分を表示できます。
Googleスプレッドシートから **ファイル →ダウンロード →カンマ区切り形式（.csv）** で書き出したものが使えます（同梱の `sample-data.csv` も同じ形式）。

## うまく表示されないとき

- **「取得に失敗しました」と出る** → スプレッドシートの共有設定が「リンクを知っている全員が閲覧可」になっているか確認してください。
- **数値が空欄（—）になる** → その列の見出し名が様式の表と一致しているか確認してください（全角/半角・スペースの違いに注意）。
- **作品セレクタが出ない** → タブ(gid)が1つしか登録されていない場合は非表示になります（正常）。

## 技術メモ

- 依存は Chart.js（CDN）のみ。ビルド不要の単一HTML。
- Googleスプレッドシートの **gviz JSON** を `<script>` タグ（JSONP）で取得するため、CORSの影響を受けず `file://` でも動作します。APIキーは不要です。
- 入力したスプレッドシートIDやタブ設定はブラウザの **localStorage** に保存されます（外部送信なし）。
- ブラウザからはスプレッドシートのタブ一覧を自動取得できないため、gidは手動で登録する方式です。
